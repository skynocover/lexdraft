# 推理策略模型比較研究報告

> 測試日期：2026-02-28
> 測試案件：艾凡尼 茱莉家 — 114年高雄法院車禍損害賠償（case_id: `z4keVNfyuKvL68Xg1qPl2`）
> 測試範圍：書狀 pipeline Step 2（推理策略），包含 Phase 1 Tool-Loop 推理 + Phase 2 JSON 結構化輸出

---

## 1. 研究背景與目的

書狀 pipeline 的 Step 2（推理策略）是整個流程中最昂貴的環節，每次執行約 $0.10–0.15 USD（Claude Haiku 4.5）。本研究旨在評估是否有更便宜的模型能在維持推理品質的前提下降低成本。

### 測試架構

Step 2 分為兩個 phase：

- **Phase 1（Tool-Loop 推理）**：模型自由推理，可呼叫 `search_law`（真實 MongoDB Atlas 法條搜尋）和 `finalize_strategy`（輸出推理摘要 + 逐爭點分析）。最多 6 輪、6 次搜尋。
- **Phase 2（JSON 結構化輸出）**：使用 Phase 1 的推理結果，生成結構化的論證策略 JSON（claims + sections）。

### 測試案件資料

| 項目 | 內容 |
|------|------|
| 案件類型 | 車禍損害賠償 |
| 書狀類型 | 民事起訴狀 |
| 爭點數 | 6 個 |
| 檔案數 | 6 份（交通事故分析研判表、診斷證明書、車鑑會鑑定意見書、損害賠償明細、在職薪資證明、調解不成立證明書） |
| 損害金額 | NT$423,700（醫療 41,550 + 交通 13,300 + 工作損失 156,000 + 機車修復 12,850 + 精神慰撫金 200,000） |
| 已有法條 | §184、§193、§195、§213、§216 |

### API 路由

所有模型均透過 Cloudflare AI Gateway 路由：

| 模型 | Gateway 路徑 | 認證方式 |
|------|-------------|---------|
| Claude Haiku 4.5 | `/{gateway}/anthropic/v1/messages` | `cf-aig-authorization` |
| DeepSeek / MiniMax / 其他 OpenRouter 模型 | `/{gateway}/openrouter/v1/chat/completions` | `cf-aig-authorization` + `cf-aig-byok-alias: lex-draft-openrouter` |

## 2. 候選模型篩選

### 初始候選（5 個模型）

| 模型 | 價格 (in/out per 1M tokens) | 篩選結果 |
|------|---------------------------|---------|
| Claude Haiku 4.5 | $0.80 / $4.00 | **Baseline** — 現行生產模型 |
| DeepSeek V3.2 | $0.25 / $0.38 | **淘汰** — 推理品質不合格 |
| Qwen 3.5 Plus | $0.20 / $0.88 | **淘汰** — model ID 問題 + 無明顯優勢 |
| Kimi K2.5 | $0.35 / $1.40 | **淘汰** — 速度過慢（385s） |
| MiniMax M2.5 | $0.295 / $1.20 | **進入決賽** |

### 淘汰原因

**DeepSeek V3.2**：成本最低（$0.004/次），但完全不呼叫 `search_law`（0 次搜尋），涵攝極度簡略（每條約 10–20 字），攻防策略空洞（「提供XX單據，證明請求金額的合理性」）。推理品質不符合法律文書要求。

**Kimi K2.5**：推理品質佳（coverage 4/6），但平均耗時 385 秒，遠超可接受範圍。生產環境中使用者無法等待 6 分鐘以上。

**GLM 5**：未實測。定價 $0.95 / $2.55，與 Claude Haiku 差距不大，無成本優勢。

## 3. 最終比較：Claude Haiku 4.5 vs MiniMax M2.5

### 3.1 效率與成本

| 指標 | Claude Haiku 4.5 | MiniMax M2.5 | 差異 |
|------|------------------|--------------|------|
| 總耗時 | 175s | 269s | MiniMax 慢 54% |
| 成本 | **$0.119** | **$0.026** | MiniMax 便宜 4.6 倍 |
| 推理輪數 | 3 rounds | 6 rounds | MiniMax 需更多輪 |
| 法條搜尋 | 6 searches | 5 searches | 接近 |
| Input tokens | 47,779 | 51,824 | MiniMax 多 8%（因多輪累積） |
| Output tokens | 20,228 | 9,333 | Claude 輸出量大 2.2 倍 |
| JSON 解析 | ✅ | ✅ | 兩者皆成功（需 jsonrepair） |
| 爭點覆蓋 | 6/6 | 6/6 | 兩者皆完整 |
| Claims 數 | 12 | 12 | 相同 |
| Sections 數 | 9 | 9 | 相同 |

> 備註：MiniMax 耗時較長的主因是推理輪數多（6 rounds vs 3 rounds）。Claude 輸出量大是因為每個爭點的涵攝和攻防論述更長。

### 3.2 推理摘要比較

**Claude Haiku 4.5**：
> 本案被告王建宏左轉未讓直行車先行，構成民法§184(1)前段過失侵權。陳美玲身體健康被不法侵害，依§193、§195、§196請求醫療費41,550元、交通費13,300元、工作損失156,000元、機車修復費12,850元及精神慰撫金200,000元。**關鍵爭點在於工作損失日數（原告3個月92日 vs 被告2個月）及精神慰撫金額度（原告200,000 vs 被告50,000）。應強調診斷書第二份記載左肩關節活動度受限持續至2025/3/28，平面設計師職業特性，雇主核准92日傷病假，及類案判決精神慰撫金範圍。**

**MiniMax M2.5**：
> 本件為交通事故損害賠償案件，被告已自認肇事責任，僅就賠償金額有所爭執。原告依民法第184條第1項前段請求侵權行為損害賠償，並就各項目分別依據：醫療費用、交通費用、不能工作損失依民法第193條第1項；財物損害依民法第196條；精神慰撫金依民法第195條第1項。**被告對於不能工作期間及精神慰撫金額度有所抗辯，原告已準備完整證據逐一反駁，整體請求應予准許。**

| 面向 | Claude | MiniMax |
|------|--------|---------|
| 具體數字引用 | 列出每項金額 + 雙方爭議焦點 | 概括描述，未列具體金額 |
| 策略重點標示 | 指出兩大關鍵爭點 + 舉證方向 | 指出爭點但未深入策略方向 |
| 可操作性 | 高（直接指出需強調哪些證據） | 中（方向正確但較泛） |

### 3.3 逐爭點分析：深度比較

#### 爭點 1：肇事責任（§184）

| | Claude Haiku 4.5 | MiniMax M2.5 |
|--|------------------|--------------|
| 請求權基礎 | §184(1)前段 + §217 | §184(1)前段 + §217 |
| 涵攝結構 | ✅ 逐要件拆解（不法侵害→過失→因果→損害→§217排除） | ✅ 逐要件拆解（過失→權利侵害→因果→損害） |
| 涵攝內容 | 詳細（引用道交§102(1)(7)、車鑑會結論） | 詳細（同樣引用道交§102(1)(7)） |
| 攻防 | 「被告已承認、車鑑會確認」 | 「被告已自認、車鑑會為證」 |
| 品質評分 | ★★★★★ | ★★★★☆ |

**分析**：兩者在此爭點的表現接近，都做到了逐要件涵攝。Claude 額外展開了 §217 過失相抵的排除論述。

#### 爭點 4：不能工作損失（§193）— 最核心爭點

| | Claude Haiku 4.5 | MiniMax M2.5 |
|--|------------------|--------------|
| 涵攝要件數 | 5 個（喪失勞動能力→損害發生→工作恢復→金額計算→職業特殊性） | 3 個（喪失勞動能力→期間→金額） |
| 職業特殊性 | ✅「平面設計需要上肢靈活操作，肩關節活動受限直接影響工作能力」 | ✅「平面設計師，需肩關節靈活度從事工作」 |
| 核心論點 | ✅ **「醫囑最低休養期」vs「完全恢復職業能力所需時間」的概念區分** | ✅ **也用了相同論點：「混淆醫囑最低休養期與完全恢復職業能力所需時間」** |
| 反算論證 | ✅「若被告堅持2個月(60日)，損失應為156,000×(60/92)≈101,739元，仍屬高額」 | ❌ 無 |
| 證據援引 | 診斷書第二份 + 雇主核准92日 + 復健記錄 | 在職薪資證明 + 實際請假92天 |
| 品質評分 | ★★★★★ | ★★★★☆ |

**分析**：Prompt 強化後 MiniMax 成功產出「醫囑最低休養期 vs 完全恢復職業能力」這個關鍵論點（上一輪測試沒有）。但 Claude 額外做了反算論證（量化差額），這在法庭上是很有說服力的技巧。

#### 爭點 6：精神慰撫金（§195）— 金額攻防最激烈

| | Claude Haiku 4.5 | MiniMax M2.5 |
|--|------------------|--------------|
| 情節重大性分析 | 5 層分析：(1)骨折重傷需手術 (2)多處外傷 (3)長期復健 (4)後遺症 (5)工作中斷92日 | 3 層分析：(1)須手術 (2)留下後遺症 (3)復健近4個月 |
| 類案行情 | ✅ 「骨折精神慰撫金範圍為80,000-300,000元，原告200,000元落在中段」 | ✅ 「涉及骨折且須手術之交通事故，精神慰撫金通常在15-30萬元區間」 |
| 職業影響 | ✅ 「平面設計師因左肩受限可能長期影響工作效率和收入」 | ✅ 「肩臂靈活度對職業至關重要」 |
| 過失程度論證 | ✅ 「被告完全過失，無過失相抵，應提高額度」 | ❌ 未提及 |
| 協商空間建議 | ✅ 「即使法院酌減亦應在160,000-180,000元以上」 | ❌ 未提及 |
| 品質評分 | ★★★★★ | ★★★★☆ |

**分析**：兩者都提供了類案行情範圍和職業影響論證（MiniMax 在 prompt 強化後成功補上）。Claude 更強的地方在於：(1) 情節重大性分析更細緻（5 層 vs 3 層）；(2) 將「被告完全過失」納入提高慰撫金的論據；(3) 提供協商底線建議。

#### 其餘爭點（醫療費、交通費、機車修復費）

| | Claude | MiniMax |
|--|--------|---------|
| 涵攝 | 每項 4–5 個要件 | 每項 3 個要件 |
| 攻防 | 具體反駁架構（3–4 點） | 較簡短（1–2 句） |
| 法條引用 | §193 + §213 + §216 | §193（部分加 §216） |
| 品質評分 | ★★★★★ | ★★★☆☆ |

**分析**：在爭議性較低的項目上，Claude 仍維持高品質輸出（每項都有完整反駁架構），MiniMax 則較為簡略但基本正確。

| | Claude Haiku 4.5 | MiniMax M2.5 |
|--|------------------|--------------|
| 搜尋次數 | 6 | 5 |
| 搜到法條數 | 17 | 15 |

## 4. Prompt 優化實驗

### 4.1 涵攝 / 攻防品質優化

在 System Prompt 的「構成要件檢視」和「攻防預判」區塊加入更明確的指引後，MiniMax 的推理品質顯著提升：

**新增指引**：
```
2. 構成要件涵攝（逐要件拆解）
   - 逐一將案件事實涵攝到各要件中，引用具體數字、日期、金額
   - 涵攝格式：要件名稱 → 對應事實 → 小結論

3. 攻防預判（具體反駁策略）
   - 引用案件中的具體數字來反駁
   - 區分概念差異（如「醫囑最低休養期」vs「完全恢復職業能力所需時間」）
   - 如有可能，提供類案判決的合理金額區間作為參考
   - 論述職業特殊性對損害程度的影響
```

**效果**：

| 維度 | 優化前 MiniMax | 優化後 MiniMax |
|------|---------------|---------------|
| 涵攝結構 | 整段敘述，未拆解要件 | ✅ 逐要件拆解（1.過失 2.權利侵害 3.因果 4.損害） |
| 類案行情 | 無 | ✅ 「骨折且須手術，精神慰撫金通常在15-30萬元」 |
| 職業影響 | 無 | ✅ 「平面設計師，需肩關節靈活度從事工作」 |
| 概念區分 | 「8週+復健=3個月」 | ✅ 「混淆醫囑最低休養期與完全恢復職業能力所需時間」 |
| 品質評級 | ~80% Claude | **~90% Claude** |

### 4.2 dispute_id 修正

**問題**：JSON schema example 中使用 `"dispute_id": "issue_1"` 作為 placeholder，模型看到後要嘛自己編 ID，要嘛留 null。真實 ID 是 nanoid（如 `Fc6FVvy_z9PPhZrpDUi2a`）。

**修正**：
1. Schema example 的 `"issue_1"` → `"（填入[爭點清單]中的真實 ID）"`
2. Section example 加上 `"dispute_id": null`（前言）和含說明的 dispute_id（內容段）
3. JSON output message 末尾加明確指示 + 插入第一個真實 ID 作為範例

**效果**：修正前兩個模型的 dispute_id 全部 null（coverage 0/6），修正後全部正確（coverage 6/6）。此修正已同步到正式 pipeline（`strategyConstants.ts` + `reasoningStrategyStep.ts`）。

### 4.3 JSON 解析修復

**問題**：A/B 腳本缺少 `jsonrepair` fallback，導致 Claude 輸出被 markdown code block 包裹時解析失敗。

**修正**：在 `tryParseJson` 加入 `jsonrepair` 作為第三層 fallback（與正式 pipeline 一致）。

**效果**：兩個模型的 JSON 解析從 50% 提升到 100% 成功率。

## 5. 技術發現

### 5.1 OpenRouter via AI Gateway 認證

AI Gateway 的 BYOK（Bring Your Own Keys）功能需要 `cf-aig-byok-alias` header 來指定 OpenRouter 存儲的供應商金鑰別名。缺少此 header 會導致 401 "No cookie auth credentials found" 錯誤。

### 5.2 回應快取問題

OpenRouter 對相同 prompt 會快取回應（第二次呼叫可能 < 1 秒返回快取結果）。解決方式：在 user message 末尾附加隨機 nonce：
```
[session: ${Date.now()}-${Math.random().toString(36).slice(2, 8)}]
```

### 5.3 搜尋噪音

兩個模型的法條搜尋都有約 50% 噪音（搜到不相關法條如 §922 典權、§1119 扶養等）。這是 MongoDB Atlas Search + smartcn tokenizer 的已知限制，與模型無關。CONCEPT_TO_LAW 改寫表可部分解決，但概念搜尋的精準度仍有改善空間。

## 6. 結論與建議

### 品質總評

| 維度 | Claude Haiku 4.5 | MiniMax M2.5 (prompt 強化後) |
|------|------------------|------------------------------|
| 涵攝完整性 | ★★★★★ | ★★★★☆ |
| 攻防策略深度 | ★★★★★ | ★★★★☆ |
| 法條搜尋 | ★★★★☆ | ★★★★☆ |
| JSON 結構 | ★★★★★ | ★★★★★ |
| 書狀語氣 | ★★★★☆ | ★★★★★ |
| 成本效益 | ★★☆☆☆ | ★★★★★ |

### Claude 仍勝出的具體差距

1. **反算論證**：Claude 會做「若被告堅持2個月，損失 = 156,000×(60/92) ≈ 101,739元」這種量化反駁，MiniMax 不會
2. **協商底線建議**：Claude 會提供「即使法院酌減亦應在 160,000–180,000 元以上」，MiniMax 不會
3. **情節重大性分級**：Claude 的分析更系統化（5 層 vs 3 層）
4. **低爭議項的論述密度**：醫療費、交通費等 Claude 仍維持 4–5 點反駁架構，MiniMax 降至 1–2 句

### 建議

**短期**：維持 Claude Haiku 4.5 作為 Step 2 模型。推理品質的差距（尤其在核心爭點的攻防深度）對法律文書品質有直接影響，目前 $0.12/次 的成本在可接受範圍。

**中期觀察**：MiniMax M2.5 以 1/4.6 的成本達到約 90% 的品質，且透過 prompt 優化仍有提升空間。若未來成本壓力增大，或 MiniMax 模型更新後品質提升，可作為替代方案。

**已落地的優化**：
- dispute_id prompt 修正 → 已同步到正式 pipeline（`strategyConstants.ts` + `reasoningStrategyStep.ts`）
- 涵攝 / 攻防 prompt 強化 → 目前僅在 A/B 腳本中，可視 Claude 品質是否需要提升再決定是否同步
